## Agent Brief

**Category:** enhancement
**Summary:** Add box and colour controls to the properties panel, including rgb→hex seeding for the colour pickers.

**Current behavior:**
After the walking skeleton (and typography controls), the panel handles type but has no spacing or colour controls. Computed colour values come back as `rgb(...)`, which native colour inputs cannot consume directly.

**Desired behavior:**
The panel exposes margin, padding, text colour, and background-colour. Colour inputs are seeded from the element's computed colour via an rgb→hex conversion, preview live on change, and record into the element's Patch. background-colour can be set without disturbing unrelated properties.

**Key interfaces:**
- An `rgbToHex(rgb)` helper converting a computed `rgb()/rgba()` string to a `#rrggbb` value for native `<input type="color">` seeding.
- The same panel control pattern as 0001/0002 - each control records a CSS property/value into the selected element's `changes` map and applies it inline.

**Acceptance criteria:**
- [ ] Panel exposes margin, padding, text colour, and background-colour controls
- [ ] Colour inputs are seeded from computed colour via rgb→hex conversion
- [ ] Changing any control previews live on the element
- [ ] Each change is recorded into the element's Patch and appears in the saved Batch
- [ ] background-colour can be set without affecting unrelated properties

**Out of scope:**
- Typography controls (issue 0002)
- Resize and position-nudge (issues 0004 / 0005)
- Borders, box-shadow, gradients (v1 property scope excludes these)
