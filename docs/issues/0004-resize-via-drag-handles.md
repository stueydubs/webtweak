## Agent Brief

**Category:** enhancement
**Summary:** Let the selected element be resized by dragging its right and bottom edges, recording width/height into the Patch.

**Current behavior:**
After the walking skeleton, an element can be selected and restyled via the panel, but it cannot be resized by direct manipulation. The vendored interact.js is available but not yet wired.

**Desired behavior:**
The selected element is resizable by dragging its right edge, bottom edge, or bottom-right corner only - the top-left origin must never move, so resizing and position-nudging stay cleanly separable. width and height update live during the drag, are recorded into the element's Patch, and the selection box (and any panel width/height fields) stay in sync. Resizing must not trigger element selection changes or page navigation.

**Key interfaces:**
- interact.js `resizable` configured on the selected element with edges limited to right and bottom (no top/left).
- The resize handler sets the element's width/height inline and records them into the `changes` map; the selection box repositions on each resize move.

**Acceptance criteria:**
- [ ] The selected element can be resized by dragging its right/bottom edges and corner
- [ ] The top-left origin does not move during resize
- [ ] width and height update live and are recorded into the element's Patch
- [ ] The selection box and any panel width/height fields stay in sync during resize
- [ ] Resizing does not change the selection or navigate the page

**Out of scope:**
- Position nudge / dragging the element (issue 0005)
- Top and left resize edges (deliberately excluded to avoid moving the origin)
