# Issue 0005: Position nudge via drag

> Labels: enhancement, ready-for-agent · Type: AFK

## What to build

Wire interact.js `draggable` onto the selected element so it can be nudged by dragging. The drag is previewed via `transform: translate(...)`, snapped to a 4px grid, and recorded into the Patch as a `{dx, dy}` nudge *intent* (not literal final CSS - Claude reconciles it into clean margin/padding). The selection box tracks the element during the drag.

## Agent Brief

**Category:** enhancement
**Summary:** Let the selected element be nudged by dragging, recording a 4px-snapped offset as intent.

**Current behavior:**
After the walking skeleton, an element can be selected and restyled, but it cannot be repositioned by direct manipulation. interact.js is vendored and (after 0004) may already be wired for resize.

**Desired behavior:**
The selected element can be dragged to a new position, previewed via `transform: translate(...)`. The offset snaps to a 4px grid and is recorded into the element's Patch as a `{dx, dy}` nudge - captured as *intent*, not literal final CSS (Claude reconciles it into clean margin/padding). The selection box tracks the element during the drag. Drag and resize coexist on the same element without conflict: edge zones resize, the interior drags.

**Key interfaces:**
- interact.js `draggable` on the selected element, accumulating dx/dy, snapping to 4px, applying via `transform: translate`.
- The drag handler records `changes.nudge = { dx, dy }`; the selection box repositions on each drag move.
- Coexistence with the resizable edges from 0004 (resize takes the edge grab-zone, drag the interior).

**Acceptance criteria:**
- [ ] The selected element can be dragged, previewed via `transform: translate`
- [ ] The offset snaps to a 4px grid
- [ ] The offset is recorded into the element's Patch as `{dx, dy}`
- [ ] The selection box tracks the element during the drag
- [ ] Drag and resize coexist on the same element without conflicting

**Out of scope:**
- Translating the nudge into final margin/padding CSS (that is the reconcile step, done by Claude)
- Resize (issue 0004)
- Structural reordering / moving an element to a different parent (v2)
